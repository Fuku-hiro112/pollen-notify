📗 References リファレンス

特定の技術・ライブラリの使い方・罠をまとめた参照資料。書いている途中で何度も戻ってくる。

  • 命名規則リファレンス（Java / JPA / DB） 2026-05-31 Phase 1

    PascalCase / camelCase / snake_case / UPPER_SNAKE の使い分け、Java↔DBの命名自動変換、テーブル・列・FK・シーケンスの命名、ファクトリ命名（of/from/valueOf）、やりがちな失敗。

  • JPA アノテーション リファレンス 2026-05-27 Phase 1

    @Entity, @Id, @GeneratedValue, @SequenceGenerator, @Column, @ManyToOne, @JoinColumn 等の用途・使用例・メリット/デメリット・本プロジェクト推奨。最小構成テンプレ付き。

📙 Design Decisions 設計判断

設計判断の経緯・選択肢・採用理由を記録するドキュメント（Architecture Decision Record 的）。後から「なぜこう決めたか」を思い出すための資料。

  • Entity 設計判断ガイド（Lombok / ID 戦略 / 日時型） 2026-05-26 Phase 1

    OOP・保守性視点から、Lombok・ID 生成戦略・日時型の選択肢を7つの判断軸で評価。SEQUENCE / Instant / @Getter+@NoArgsConstructor(PROTECTED) を推奨。

📕 Plans プラン

特定の作業（修正・実装・リファクタ）の手順チェックリスト。完了すれば役目を終える。

  • Entity 修正プラン（ObservationPoint / PollenMeasurement） 2026-05-30 Phase 1

    DDL 逸脱した ObservationPoint の修正と、PollenMeasurement 新規実装のチェックリスト。検証手順・よくあるエラー例も収録。
