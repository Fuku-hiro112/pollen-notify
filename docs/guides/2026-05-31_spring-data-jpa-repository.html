3. 最小実装：インターフェースを継承するだけ

クラスではなく インターフェース を作り、JpaRepository<T, ID> を継承します。型引数は2つ：

型引数意味ObservationPoint の場合
T対象の Entity 型ObservationPoint
IDその Entity の主キーの型Long（id が Long なので）
🔵 形（ObservationPoint 用） 
package com.fuku.pollen_notify.repository;

import com.fuku.pollen_notify.entity.ObservationPoint;
import org.springframework.data.jpa.repository.JpaRepository;

public interface ObservationPointRepository
        extends JpaRepository<ObservationPoint, Long> {
    // 中身は空でも動く（次章のメソッドはあとで足す）
}
🖊️ あなたが書くもの：repository パッケージを新規作成し、上を参考に 2つのインターフェースを自分で書く。
  • ObservationPointRepository … JpaRepository<ObservationPoint, Long>
  • PollenMeasurementRepository … JpaRepository<PollenMeasurement, ___>（主キーの型は自分で確認して埋める）

4. タダで手に入るメソッド

JpaRepository を継承した時点で、1行も書かずにこれらが使えます（抜粋）。

メソッド用途
save(entity)INSERT または UPDATE（保存）
findById(id)主キーで1件取得 → Optional<T>
findAll()全件取得
existsById(id)存在チェック
deleteById(id)削除
count()件数
🟣 findById が Optional<T> を返すのが重要。「見つからないかもしれない」を型で表現し、null チェック忘れを防ぐ設計です。Optional の扱いは Phase 1 で慣れておくと後が楽。

5. クエリメソッド命名規約（派生クエリ）

標準メソッドにないものは、メソッド名を規約どおりに書くだけで Spring が SQL を自動生成します（派生クエリ / derived query）。

キーワードメソッド名の例生成される条件
findByfindByExternalId(String externalId)WHERE external_id = ?
existsByexistsByExternalId(String id)存在するか（boolean）
countBycountByObservationPoint(...)件数
And / OrfindByAAndBB(...)WHERE a = ? AND b = ?
OrderBy ... DescfindTopByXxxOrderByMeasuredAtDesc(...)並べ替えて先頭1件
Top / FirstfindTop10By...件数制限
🟡 注意：プロパティ名は「Entity のフィールド名（camelCase）」で書く。DB列名（snake_case）ではない。
例：findByExternalId（◯ フィールド externalId）／ findByExternal_id（✕）。
※ 命名規則ガイドの「Java↔DBの変換」と同じ理屈。メソッド名はJava世界なのでcamelCase。
🔵 複雑になったら @Query で JPQL/SQL を直接書く逃げ道もあります（Phase 3以降）。Phase 1は派生クエリで十分。
📚 公式：Spring Data JPA – Query Methods

7. @Repository は要る？

🟢 結論：付けなくても動く。JpaRepository を継承したインターフェースは Spring が自動で Bean として登録します。@Repository を付けても害はない（明示の意味で付ける流派もある）。このプロジェクトでは省略でOK。

8. やりがちな失敗

🔴 class で作ってしまう：Repository は interface。class にすると「実装しろ」と怒られる。実装はSpring任せ。
🔴 ID の型を間違える：JpaRepository<ObservationPoint, Long> の2つ目は主キーの型。id が Long なら Long。int や Integer ではない。
🟡 メソッド名にDB列名（snake_case）を書く：findByExternal_id ではなく findByExternalId（フィールド名）。

9. 理解度チェック

Q1. Repository は class と interface どっち？ 実装は誰が書く？

interface。実装は Spring Data JPA が起動時に自動生成する。自分では書かない。

Q2. JpaRepository<PollenMeasurement, ?> の ? に入る型は？

Long。PollenMeasurement の id フィールドが Long 型だから（主キーの型を指定する）。

Q3. 「external_id が一致する地点を1件」取るメソッド名は？ 戻り値の型は？

findByExternalId(String externalId)。戻り値は Optional<ObservationPoint>（見つからない可能性を型で表す）。

Q4. save() や findById() は自分で書く必要がある？

不要。JpaRepository を継承した時点で標準装備される。
